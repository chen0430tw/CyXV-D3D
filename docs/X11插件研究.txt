# 1️⃣ X11 opcode 的整体结构

X11 request header 基本格式：

```
byte 0   major opcode
byte 1   minor opcode
byte 2-3 length
```

其中：

```
major opcode = 请求类型
minor opcode = 扩展内部操作
```

核心协议使用：

```
0 – 127
```

扩展使用：

```
128 – 255
```

也就是说：

```
128-255 是 extension opcode
```

这是 X11 协议设计时就保留的范围。 ([x.org][1])

---

# 2️⃣ Extension 的 opcode 分配方式

关键点：

**扩展不会写死 opcode。**

流程是：

### Step 1

client 查询扩展

```
QueryExtension
```

例如：

```
QueryExtension("MIT-SHM")
```

服务器返回：

```
major_opcode
first_event
first_error
```

例如：

```
major_opcode = 143
first_event = 65
first_error = 128
```

---

### Step 2

client 之后发送请求

```
major_opcode = extension opcode
minor_opcode = extension command
```

例如：

```
major = 143
minor = 0
```

表示：

```
MIT-SHM Attach
```

---

# 3️⃣ extension 内部 opcode 设计

每个 extension 自己定义 **minor opcode**。

例如：

### MIT-SHM

```
0  ShmAttach
1  ShmDetach
2  ShmPutImage
3  ShmGetImage
```

### XInput

```
0  GetExtensionVersion
1  ListInputDevices
2  OpenDevice
...
```

所以结构是：

```
major opcode  → extension id
minor opcode  → extension command
```

这就是 **X11 API 的 opcode 分层结构**。 ([x.org][2])

---

# 4️⃣ extension 注册流程（server 侧）

Xorg server 代码里通常这样注册：

```c
ExtensionEntry *AddExtension(
    char *name,
    int num_events,
    int num_errors,
    DispatchProcPtr dispatch,
    DispatchProcPtr sdispatch,
    ResetProcPtr reset,
    StandardMinorOpcodeProc minorop
);
```

返回：

```
extension->base
```

这个就是：

```
major opcode
```

服务器在 runtime 分配。

假设 Phoenix extension：


请求：

```

```

包结构：

```
byte 0  150     (major)
byte 1  0       (minor)
byte 2  length
byte 4  window
byte 8  shm_id
byte 12 frame_id
```
